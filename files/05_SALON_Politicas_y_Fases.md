## PARTE 1: POLÍTICAS LEGALES

### 1. POLÍTICA DE PRIVACIDAD

```markdown
# POLÍTICA DE PRIVACIDAD - SALON

**Última actualización:** Noviembre 2025
**Vigencia:** A partir de la fecha de publicación

## 1. INFORMACIÓN QUE RECOPILAMOS

### 1.1 Información que proporcionas directamente:
- Datos de cuenta: nombre, correo electrónico, contraseña
- Datos de perfil: foto, teléfono, fecha de nacimiento
- Datos de pago: información de tarjeta (tokenizada vía Stripe)
- Comunicaciones: mensajes de chat, reseñas, comentarios

### 1.2 Información recopilada automáticamente:
- Datos de uso: páginas visitadas, clics, tiempo de sesión
- Datos del dispositivo: IP, navegador, sistema operativo
- Ubicación: con tu consentimiento, para encontrar servicios cercanos
- Cookies: para mejorar la experiencia y recordar preferencias

## 2. CÓMO USAMOS TU INFORMACIÓN

- Prestar y mejorar nuestros servicios
- Procesar reservas y pagos
- Comunicarnos contigo (confirmaciones, recordatorios)
- Personalizar tu experiencia
- Cumplir con obligaciones legales
- Prevenir fraude y garantizar seguridad

## 3. COMPARTIR INFORMACIÓN

NO vendemos tus datos personales. Compartimos información solo con:
- Profesionales de belleza: para procesar tu reserva
- Proveedores de servicios: pagos (Stripe), emails (SendGrid)
- Autoridades: cuando lo requiera la ley
- Transferencias empresariales: en caso de fusión o adquisición

## 4. TUS DERECHOS (LEY 8968 / GDPR)

- Acceso: ver qué datos tenemos sobre ti
- Rectificación: corregir datos incorrectos
- Eliminación: eliminar tu cuenta y datos
- Portabilidad: exportar tus datos
- Objeción: oponerte al procesamiento de datos
- Restricción: limitar cómo usamos tus datos

**Para ejercer tus derechos:** privacidad@salon.com

## 5. SEGURIDAD

Implementamos medidas técnicas y organizativas:
- Encriptación SSL/TLS
- Contraseñas hasheadas (bcrypt)
- Tokenización de pagos
- Acceso restringido a datos
- Auditorías de seguridad regulares

## 6. RETENCIÓN DE DATOS

- Datos de cuenta: mientras esté activa + 30 días tras eliminación
- Datos de transacciones: 7 años (requisito legal fiscal)
- Logs de seguridad: 2 años
- Cookies: según configuración del navegador

## 7. COOKIES

Usamos cookies para:
- Esenciales: funcionamiento del sitio (no requieren consentimiento)
- Analíticas: entender uso del sitio (Google Analytics)
- Publicidad: mostrar anuncios relevantes

**Puedes gestionar cookies** en configuración del navegador.

## 8. MENORES DE EDAD

Nuestro servicio no está dirigido a menores de 18 años.
Si detectamos datos de menores, los eliminamos inmediatamente.

## 9. CAMBIOS A ESTA POLÍTICA

Notificaremos cambios significativos vía email.
Cambios menores se publicarán en esta página.

## 10. CONTACTO

**Responsable de Datos:**
- Email: privacidad@salon.com
- Teléfono: +506 XXXX-XXXX
- Dirección: [Dirección física Costa Rica]

**Autoridad de Protección de Datos (Costa Rica):**
PRODHAB - www.prodhab.go.cr
```

### 2. TÉRMINOS Y CONDICIONES DE USO

```markdown
# TÉRMINOS Y CONDICIONES - SALON

## 1. ACEPTACIÓN DE TÉRMINOS

Al usar Salon, aceptas estos términos. Si no estás de acuerdo, no uses el servicio.

## 2. DEFINICIONES

- **"Salon"**: la plataforma web/móvil
- **"Usuario"**: cualquier persona que use el servicio
- **"Cliente"**: usuario que reserva servicios
- **"Profesional"**: estilista que ofrece servicios
- **"Sucursal"**: establecimiento registrado

## 3. USO DEL SERVICIO

### 3.1 Para CLIENTES:
- Reservar servicios con profesionales
- Pagar online de forma segura
- Dejar reseñas honestas
- Cumplir con citas o cancelar con tiempo

### 3.2 Para PROFESIONALES:
- Ofrecer servicios de calidad
- Mantener disponibilidad actualizada
- Confirmar citas a tiempo
- Cumplir con estándares profesionales

### 3.3 Para ADMINISTRADORES:
- Gestionar su sucursal
- Pagar suscripción mensual
- Cumplir con normativa local
- Mantener información veraz

## 4. REGISTRO Y CUENTA

- Debes ser mayor de 18 años
- Información veraz y actualizada
- Mantener contraseña segura
- Una cuenta por persona/negocio
- No compartir credenciales

## 5. PAGOS Y FACTURACIÓN

### 5.1 Clientes:
- Pagos procesados por Stripe/PayPal
- Precios incluyen impuestos aplicables
- Comprobante electrónico vía email

### 5.2 Sucursales:
- Suscripción mensual según plan elegido
- Comisión del 7% por transacción
- Facturación automática
- Impuestos según legislación local

## 6. CANCELACIONES Y REEMBOLSOS

### 6.1 Política de Cancelación:
- Cliente puede cancelar hasta 24h antes: reembolso 100%
- Cancelación 12-24h antes: reembolso 50%
- Cancelación <12h: sin reembolso
- No-show: sin reembolso

### 6.2 Excepciones:
- Emergencias médicas (con comprobante)
- Cancelación por parte del profesional
- Problemas técnicos de la plataforma

## 7. PROPIEDAD INTELECTUAL

- Salon es propietario del software y marca
- Contenido generado por usuarios les pertenece
- Al publicar, otorgas licencia de uso a Salon
- No uses la plataforma para violar derechos de autor

## 8. CONTENIDO PROHIBIDO

No está permitido:
- Contenido ilegal, ofensivo o inapropiado
- Spam o publicidad no autorizada
- Información falsa o engañosa
- Suplantación de identidad
- Acoso o discriminación

## 9. RESPONSABILIDADES Y LIMITACIONES

### 9.1 Salon NO es responsable de:
- Calidad del servicio prestado por profesionales
- Disputas entre clientes y profesionales
- Daños físicos o materiales durante el servicio
- Reacciones alérgicas a productos

### 9.2 Limitación de Responsabilidad:
- Máximo: monto de la transacción
- No cubre daños indirectos o lucro cesante

## 10. SUSPENSIÓN Y TERMINACIÓN

Podemos suspender/terminar cuentas por:
- Violación de términos
- Actividad fraudulenta
- Reseñas falsas o manipulación
- Conducta inapropiada
- Impago de suscripción

## 11. MODIFICACIONES

- Podemos modificar términos con previo aviso
- Notificaremos vía email 30 días antes
- Uso continuado implica aceptación

## 12. LEY APLICABLE Y JURISDICCIÓN

- Ley de Costa Rica
- Tribunales de San José, Costa Rica
- Idioma oficial: Español

## 13. CONTACTO

soporte@salon.com
+506 XXXX-XXXX
```

### FASE 1: PLANIFICACIÓN (Detalle)

**Duración:** 2 meses
**Equipo:** Product Manager, UX/UI Designer, Arquitecto de Software

#### Semana 1-2: Análisis de Requisitos
```
├─ Investigación de mercado
│  └─ Análisis de competidores (Fresha, Booksy)
├─ Definición de personas (user personas)
├─ User stories por rol
├─ Priorización de features (MoSCoW)
└─ Documento PRD (Product Requirements Document)
```

#### Semana 3-4: Arquitectura Técnica
```
├─ Diagrama de arquitectura de sistema
├─ Selección de stack tecnológico
├─ Diseño de base de datos (ERD)
├─ Definición de APIs
└─ Plan de seguridad y compliance
```

#### Semana 5-6: Diseño UX
```
├─ Wireframes low-fidelity
├─ User flows por rol
├─ Mapa de navegación
├─ Pruebas de usabilidad
└─ Iteración basada en feedback
```

#### Semana 7-8: Diseño UI
```
├─ Design system (colores, tipografía)
├─ Componentes reutilizables
├─ Mockups high-fidelity (Figma)
├─ Prototipo interactivo
└─ Guía de estilo
```

### FASE 2: DESARROLLO MVP

**Duración:** 3 meses (6 sprints de 2 semanas)
**Equipo:** 2 Full-stack developers, 1 QA

#### Sprint 1-2: Autenticación y Usuarios
```
Features:
├─ Registro de usuarios (todos los roles)
├─ Login/Logout
├─ Recuperación de contraseña
├─ Perfiles básicos
└─ OAuth (Google, Facebook)

Criterios de Aceptación:
├─ Usuario puede registrarse con email
├─ Validación de campos
├─ Emails de verificación
├─ Sesiones seguras (JWT)
└─ Tests unitarios >80% cobertura
```

#### Sprint 3-4: Catálogo de Servicios
```
Features:
├─ CRUD de servicios (Admin)
├─ Listado de servicios (Cliente)
├─ Filtros y búsqueda
├─ Detalle de servicio
└─ Categorías de servicios

Tech:
├─ API REST endpoints
├─ Paginación
├─ Caching con Redis
└─ Optimización de imágenes
```

#### Sprint 5-6: Sistema de Reservas
```
Features:
├─ Calendario de disponibilidad
├─ Reservar cita
├─ Confirmar/Cancelar cita
├─ Notificaciones email
└─ Vista de citas (todos los roles)

Complejidad:
├─ Lógica de slots disponibles
├─ Prevenir doble reserva
├─ Validación de horarios
└─ Estados de cita (máquina de estados)
```

#### Sprint 7-8: Pagos
```
Features:
├─ Integración Stripe
├─ Checkout seguro
├─ Guardar métodos de pago
├─ Historial de pagos
└─ Reembolsos

Testing Crítico:
├─ Escenarios de pago exitoso
├─ Pagos fallidos
├─ Webhooks de Stripe
└─ Seguridad PCI DSS
```

#### Sprint 9-10: Dashboards
```
Features:
├─ Dashboard Cliente
├─ Dashboard Estilista
├─ Dashboard Admin Sucursal
├─ Métricas básicas
└─ Gráficas (Chart.js)

KPIs a mostrar:
├─ Citas completadas
├─ Ingresos
├─ Rating promedio
└─ Próximas citas
```

### FASE 3: FEATURES AVANZADAS

**Duración:** 4 meses (12 sprints)

#### Sprint 11-22: Features Completas
```
├─ Portafolio de estilistas (posts tipo Instagram)
├─ Chat en tiempo real (Socket.io + Firestore)
├─ Sistema de reseñas y ratings
├─ Analytics avanzado con exportación
├─ Facturación electrónica (API Hacienda)
├─ PWA: Service Worker + Cache offline
├─ Notificaciones push
├─ Multi-idioma (i18n)
├─ Temas personalizados
├─ Promociones y descuentos
├─ Programa de fidelidad
└─ Panel Super Admin
```

### FASE 4: TESTING Y LANZAMIENTO

**Mes 10: QA Exhaustivo**
```
Testing Types:
├─ Unitario (Jest, PHPUnit)
├─ Integración (Postman, Insomnia)
├─ E2E (Cypress, Selenium)

Bug Tracking:
└─ Prioridad: Critical → High → Medium → Low
```

**Mes 11: Beta Privada**

```
Beta Testers:
├─ 5 sucursales seleccionadas

└─ 50 clientes

Objetivos:
├─ Validar flujos reales
├─ Identificar edge cases
├─ Recopilar feedback
└─ Ajustar UX
```

**Mes 12: Lanzamiento**

```
Pre-launch:
├─ Marketing (landing page, redes sociales)

├─ Soporte 24/7 primera semana

Go-live Checklist:
☐ SSL configurado
☐ Backups automatizados

☐ DNS apuntando correctamente
☐ Email transaccional probado
☐ Pagos en modo producción
☐ Logs centralizados

☐ Equipo de soporte preparado
```
